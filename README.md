# Snhu-Computer-science-portfolio
Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?
The client for this course was The Gaming Room, a company that wanted to expand their game Draw It or Lose It from an Android-only application to a cross-platform solution. The client required a scalable, distributed software design that could support multiple users, teams, and game sessions simultaneously across different operating systems. The primary goal was to design a system that allowed efficient memory usage, centralized storage of image assets, secure networking, and consistent performance across platforms.
What did you do particularly well in developing this documentation?
One area that went particularly well was analyzing system architecture and making platform recommendations. Creating the software design document helped clarify how operating systems, memory management, storage, and networking all interact within a real-world application. I was able to clearly explain technical concepts in a way that would be understandable to both developers and non-technical stakeholders, which is an important skill in professional software development.
What about the process of working through a design document did you find helpful when developing the code?
Working through the design document before focusing on implementation was very helpful. The process of documenting requirements, constraints, and architectural decisions provided a strong foundation for understanding how the application should function before any code was written. This approach reduced ambiguity, helped identify potential issues early, and reinforced the importance of planning before development.
If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
If I could revise one part of my work, I would improve the level of detail in the distributed systems section. While the overall architecture was sound, adding diagrams or more detailed explanations of data flow between clients and servers would make the document even clearer. Improving this section would strengthen communication with future developers who might work on the project.
How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?
Interpreting user needs was central to the design of Draw It or Lose It. The client needed fast image rendering, support for multiple concurrent games, and cross-platform compatibility. By focusing on these needs, I was able to recommend solutions such as cloud storage, efficient memory usage, and a Linux-based server environment. Considering user needs is critical because software that does not align with client expectations or user experience goals is unlikely to succeed, regardless of how well it is coded.
How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?
In the future, I will continue to approach software design by starting with clear requirements analysis and system architecture planning. Techniques such as modular design, scalability planning, and early consideration of security and performance will guide my process. I would also use iterative reviews and documentation updates to ensure the design remains aligned with evolving requirements when designing similar software applications.
